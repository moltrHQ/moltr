Erstelle ein Docker-Setup fuer Moltr Security als standalone Produkt.
Nur der Security-Layer, NICHT der Telegram-Agent.

1. Erstelle einen neuen Ordner: C:\Users\Administrator\moltr-security\

2. Kopiere NUR die Security-relevanten Dateien rein:
   - src/moltr-client.ts (Panic Patterns + Security API Client)
   - Die Moltr Security API selbst (der Service auf Port 8420)
   - skills-registry.yaml
   - Beispiel domains.yaml (allowlist)
   - Beispiel paths.yaml (erlaubte Pfade)

3. Erstelle ein Dockerfile:
   - Base: node:20-alpine
   - Exponiert Port 8420
   - Startet den Security-Service
   - Healthcheck eingebaut

4. Erstelle docker-compose.yml:
   - Ein Service: moltr-security
   - Volume-Mount fuer Config-Dateien (domains.yaml, paths.yaml)
   - Environment Variables fuer Telegram Alert Token (optional)
   - Restart: unless-stopped

5. Erstelle eine INSTALL.md:
   - 3 Schritte: Clone, Configure, Run
   - Beispiel fuer OpenClaw Integration
   - Beispiel fuer beliebigen Agent
